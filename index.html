Automated Financial Data Pipeline

Full-stack system to automatically collect, process, and deliver personalized financial updates via backend automation and a lightweight React web interface.

‚Üì
FastAPI Python PostgreSQL SQLAlchemy Pydantic Finnhub API AWS SES AWS S3 JWT Docker GitHub Actions React

Separation of Concerns (SoC) & Layered Architecture

The backend was strictly divided into layers to keep services decoupled from the ORM.

  • Routers handled HTTP routing and request/response models.
  • Services implemented business logic (e.g., password hashing, ETL orchestration).
  • Repositories encapsulated database access logic (CRUD operations).

Asynchronous Programming for Concurrent Operations

Used asyncio.gather for concurrent fetching of stock quotes and company profiles, optimizing the Extract/Transform phase of the ETL flow. Used asyncio.to_thread to prevent blocking during synchronous API calls.

Orchestration and Observability

Implemented scheduling and logging for robust pipeline execution.

  • A GitHub Actions workflow (daily_email.yml) orchestrates the daily data dispatch.
  • Pipeline summary logs are uploaded to AWS S3 for auditing and debugging.

Python Redis From Scratch

A high-fidelity implementation of a Redis server in Python, featuring a multi-threaded, non-blocking architecture adhering to the Redis Serialization Protocol (RESP).

‚Üì
Python Multi-threading Socket Programming RESP Geo-Spatial Replication

Multi-threaded, Non-Blocking Concurrency

The server was designed to handle multiple clients simultaneously by spawning a dedicated thread for each connection using app/main.py.

  • Thread safety for shared data was managed via threading.Lock in the core data store.
  • Blocking list operations (BLPOP) were implemented using threading.Condition for efficient timed waits.

Implementing Advanced Data Structures and Replication

Implemented complex Redis features from scratch, providing a deep understanding of database internals:

  • Geo-Spatial indexing uses Morton Geohashing and distance calculation utilizes the Haversine formula.
  • Replication supports master‚Äìreplica handshake, command propagation, and durability verification.
  • Sorted Sets maintain order first by score, then lexicographically.

Lessons Learned (Low-Level Engineering)

Gained proficiency in low-level socket I/O, byte-level RESP protocol parsing, and designing thread synchronization mechanisms for a distributed system.

Data Warehouse and Analytics Project

A comprehensive data warehousing and analytics solution demonstrating industry best practices by building an ELT pipeline and generating insights from consolidated sales data.

‚Üì
SQL Server SQL ELT Pipelines Data Modeling Medallion Architecture Star Schema

Medallion Architecture Implementation

Implemented the data architecture using a layered approach for clear progression of data quality:

  • Bronze Layer: Raw data ingestion from CSV source systems.
  • Silver Layer: Cleansing, standardization, and normalization to prepare data for analysis.
  • Gold Layer: Business-ready data modeled into a star schema for reporting.

Data Modeling for Analytics (Star Schema)

The Gold Layer models data into a star schema for optimized analytical queries, enabling insights into customer behavior, product performance, and sales trends. This approach consolidates sales data from two disparate sources (ERP and CRM).

Quality Assurance Focus

A key specification was the cleansing and resolution of data quality issues in the Silver layer prior to any analysis or downstream reporting.

Sunkist Dental Clinic Website (React SPA)

A modern, responsive Single-Page Application (SPA) built for a local dental practice focusing on user experience, professional design fidelity, and a maintainable component architecture.

‚Üì
ReactJS Tailwind CSS JavaScript (ES6+) Firebase

Component-Based Architecture and SRP

The UI was broken into reusable components (e.g., Employee.js, Testimonials.js) following the Single Responsibility Principle (SRP) for modularity.

Components were also categorized into atoms and molecules following Atomic Design principles for long-term consistency.

Design-First and Responsive Implementation

Used a Design-First Approach by prototyping the UI in Figma to ensure pixel-perfect fidelity when translated to React.

Adopted a mobile-first approach using the utility-first framework Tailwind CSS to guarantee responsive design across all screen sizes and minimize CSS bloat.

Outcome and Efficiency

The solution included an online appointment booking process that successfully reduced front desk workload.
