Separation of Concerns (SoC) & Layered Architecture

The backend was strictly divided into layers to keep services decoupled from the ORM.

  • **Routers** handled HTTP routing and request/response models.
  • **Services** implemented business logic (e.g., password hashing, ETL orchestration).
  • **Repositories** encapsulated database access logic (CRUD operations).

Asynchronous Programming for Concurrent Operations

Used `asyncio.gather` for concurrent fetching of stock quotes and company profiles, optimizing the Extract/Transform phase of the ETL flow. Used `asyncio.to_thread` to prevent blocking during synchronous API calls.

Orchestration and Observability

Implemented scheduling and logging for robust pipeline execution.

  • A GitHub Actions workflow (`daily_email.yml`) orchestrates the daily data dispatch.
  • Pipeline summary logs are uploaded to **AWS S3** for auditing and debugging.

Multi-threaded, Non-Blocking Concurrency

The server was designed to handle multiple clients simultaneously by spawning a dedicated thread for each connection using `app/main.py`.

  • Thread safety for shared data was managed via `threading.Lock` in the core data store.
  • Blocking list operations (`BLPOP`) were implemented using `threading.Condition` for efficient timed waits.

Implementing Advanced Data Structures and Replication

Data Warehouse and Analytics Project

‚Üì

Medallion Architecture Implementation

Implemented the data architecture using a layered approach for clear progression of data quality:

  • **Bronze Layer**: Raw data ingestion from CSV source systems.
  • **Silver Layer**: Cleansing, standardization, and normalization to prepare data for analysis.
  • **Gold Layer**: Business-ready data modeled into a star schema for reporting.

Data Modeling for Analytics (Star Schema)

The Gold Layer models data into a star schema for optimized analytical queries, enabling insights into customer behavior, product performance, and sales trends. This approach consolidates sales data from two disparate sources (ERP and CRM).

Quality Assurance Focus

A key specification was the cleansing and resolution of data quality issues in the Silver layer prior to any analysis or downstream reporting.
